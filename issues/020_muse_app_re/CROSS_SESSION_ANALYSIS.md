## OSCパケットロス問題

設定を変えていないにもかかわらず、見かけ上のサンプリングレートが最大4倍異なる。

| 指標 | Session1 | Session2 | Session3 |
|------|----------|----------|----------|
| Museハードウェアレート | 256 Hz | 256 Hz | 256 Hz |
| 見かけの実効レート | ~51.5 Hz | ~206.5 Hz | ~191.2 Hz |
| データのある秒 / 全秒 | 306 / 899 (**34%**) | 505 / 601 (**84%**) | 964 / 1213 (**79%**) |

- Session1は**66%の秒でデータが全く届いていない**（大規模パケットロス）
- Session2, 3はほぼ安定（パケットロス軽微）

> **結論**: Session1はパケットロスによる影響が大きく、S2/S3とは別扱いが必要。

## 前処理

- バンドパワー: Welch PSD (2s窓, 1sステップ) → Alpha(8-13Hz), Beta(13-25Hz)積分
- チャンネル: 前頭部(AF7, AF8)平均
- ダウンサンプル: アンチエイリアスフィルタ付きで51.5Hz相当に統一
- 指標: **Alpha/Beta リニア比** → 4秒中心移動平均
- Z-score正規化: **(ratio - calib_mean) / calib_std**（キャリブレーション期間で正規化）
- キャリブレーション: **最初の60秒**（アプリのグラフが1:00から開始）

## 1. Mind State 分類

### 推定アルゴリズム

```
指標:   前頭部(AF7, AF8)の Alpha(8-13Hz) / Beta(13-25Hz) パワー比
手法:   Welch PSD (2s窓) → バンド積分 → リニア比 → 4秒中心移動平均
基準:   キャリブレーション期間(60s)の mean, std (Z-score正規化)

分類:
  Calm    = Z-score ≥ zc
  Active  = Z-score ≤ -za
  Neutral = その間
```

### 個別最適パラメータ

| Session | calm_z (zc) | active_z (-za) | err |
|---------|-------------|----------------|-----|
| S1 | -0.15 | -1.90 | **61s** |
| S2 | -1.15 | -2.30 | **8s** |
| S3 | +0.75 | -1.60 | **16s** |

- S2, S3は高精度（誤差8-16秒）
- S1はパケットロスの影響で誤差が大きい
- **calm_z は セッションごとに大きく異なる**（-1.15〜+0.75）
- **active_z は比較的安定**（-1.60〜-2.30 の範囲）

### 共通パラメータの限界

3セッション共通パラメータ（Z-score: calm_z=-0.05, active_z=-1.70）のtotal_err=643s。
S3はZ-score中央値が+2.91であり、calm_z=-0.05を適用するとほぼ全区間がCalmになってしまう。

> **結論**: Mind State分類にはセッション固有の適応的閾値が必要。
> "active_z"は-1.6〜-1.9付近で安定しているが、"calm_z"はセッション分布に応じた調整が必要。

### 確信度: **中**（個別セッションは高、汎化は困難）

## 2. Birds 検出

### 最有力仮説: 仮説D (Calm内 N秒/bird)

```
Birds = sum( floor(calm_segment_duration / base_sec) )
        ※各Calmセグメントで、base_sec秒ごとに1 Bird
```

### 個別最適パラメータ

| Session | base_sec | 推定Birds | 実際 | err |
|---------|----------|-----------|------|-----|
| S1 | 15.5s | 5 | 5 | 0 |
| S2 | **8.5s** | 29 | 29 | 0 |
| S3 | **8.5s** | 88 | 87 | **1** |

**Session2とSession3が完全に同じbase=8.5sで一致**。これは強い証拠。

### セッション間の比較

| Session | Calm秒 | Birds | Calm/Birds比 |
|---------|--------|-------|--------------|
| S1 | 369s | 5 | 73.8s/bird |
| S2 | 403s | 29 | 13.9s/bird |
| S3 | 890s | 87 | 10.2s/bird |

- S2/S3では Calm時間の分割方法が非常に近い（8.5〜10s/bird）
- S1の73.8s/birdは極端に大きい → パケットロスによるCalm分布の歪みが原因と推定

### 共通パラメータ (3セッション)

| 方式 | base_sec | total_err | 詳細 |
|------|----------|-----------|------|
| Z-score共通Calm + base | 10.0s | 28 | S1=10/t=5, S2=9/t=29, S3=90/t=87 |
| MAD共通Calm + base | 12.0s | 21 | S1=12/t=5, S2=17/t=29, S3=89/t=87 |

S3はどの方式でも精度が高い (err=1〜3)。S2の誤差が大きい。

### 暫定的な推定

```
1. 個別最適ではbase=8.5s (S2, S3で完全一致)
2. Birds ≈ floor(Calm合計秒 / 8.5s) 程度が最も有力
3. S1のみ15.5s → パケットロスによる分布変化で見かけの単価が上昇
4. Calmセグメントの分割も重要（連続Calm内の計数方式）
```

### 確信度: **中〜高**

- S2/S3の完全一致（base=8.5s）は強い証拠
- S1のパケットロス影響を除けば、base≒8.5-10sが安定

## 3. Recoveries 検出

### 推定アルゴリズム

```
定義:   Alpha/Beta比の局所的な上昇（momentary dip → recovery）
手法:   前後ウィンドウのZ-score平均を比較

条件:
  - 直前 lb=3秒の平均 Z-score ≤ drop_z
  - 直後 lf=3秒の平均 Z-score ≥ rise_z  (rise_z > drop_z)
  - 最小イベント間隔: 30秒
```

### 個別最適パラメータ

| Session | lb | lf | drop_z | rise_z | 検出数 | err |
|---------|----|----|--------|--------|--------|-----|
| S1 | 3s | 3s | -1.2 | +0.3 | 3 | 0 |
| S2 | 3s | 3s | -1.8 | -0.5 | 3 | 0 |
| S3 | 3s | 3s | -0.6 | -0.5 | 8 | 0 |

**全セッションで個別精度は完全一致 (err=0)**。

### 共通パラメータの困難

3セッション共通パラメータ（lb=3s, lf=7s, drop=0.0, rise=1.1）のtotal_err=6。

Session3のZ-score分布の大きな違いが原因:
- S1/S2: 瞑想期間のZ-score中央値 ≈ -0.2〜-0.4（キャリブより低め）
- S3: 瞑想期間のZ-score中央値 ≈ +2.9（キャリブより大幅に高め）

S3では「一時的なdip」がS1/S2の絶対値より高い位置で発生するため、
S1/S2のdrop_z=-1.6のような低い閾値では一切検出されない（err=8）。

### 観察された一貫性

- **lb=lf=3s**が全セッションで最適 → この時間スケールは確定的
- drop_z と rise_z は**セッションのZ-score中央値に相対的**に決まる
- **rise_z は drop_z より常に高い**（回復方向の条件）

### 確信度: **中**

- lb=lf=3s の時間スケールは確定的
- 絶対的なZ-score閾値はセッション分布依存
- セッション内中央値に対する相対閾値での再定式化が有望

## Birds vs Recoveries の関係

| | Recoveries | Birds |
|--|------------|-------|
| 検出対象 | **局所的な回復**（dip→calm遷移） | **Calm時間の累積** |
| 時間特性 | 瞬間的（3s前後窓） | 持続的（Calm秒数に比例） |
| Calm%との関係 | **弱い** | **強い（ほぼ線形）** |
| 意味 | 何回注意を取り戻したか | どれだけ長く集中を維持したか |

```
Session2: Active=5s, Calm=403s (73%) → Recoveries=3, Birds=29
Session3: Active=5s, Calm=890s (76%) → Recoveries=8, Birds=87
```

- **Birdsはほぼ Calm時間に比例**（8.5〜10s/bird）
- **Recoveriesは非線形**（S2=S3同じActive秒なのにRecoveriesが異なる）
